# 三维线条切割 CAM 产品规格文档

## 1. 文档定位

本文定义 `laser-3d-cam` 产品的产品级规格，用于确认产品要做什么、做到什么程度、如何验收。

本文不讨论具体工程实现，不规定必须使用某个几何内核、渲染引擎或 UI 技术。实现方案以同目录下的方案文档为准。

## 2. 产品定位

产品名称：

```text
三维线条切割 CAM
```

产品 ID：

```text
icax.laser-3d-cam
```

产品定位：

- 面向三维 CAD 模型上的线条切割 CAM。
- 用户导入三维 CAD 模型后，通过自动特征识别和手动拾取获得切割刀路。
- 切割刀路以三维空间曲线表达。
- 刀路可以叠加工艺，如微联、引线、引出。
- 系统可以规划刀路加工顺序，尽量减少空移。
- 系统可以基于切割头机械模型做 CAM 仿真，并进行碰撞检测，已切割路径显示为红色。

## 3. MVP 范围

MVP 目标是完成从 CAD 模型到可仿真的切割路径闭环。

MVP 主流程：

```text
导入五轴切割头/刀具描述文件
  -> 校验刀具子部件、关节、TCP 和必要参数
  -> 显示并驱动切割头模型
导入三维 CAD 模型
  -> 显示模型
  -> 自动识别特征或手动拾取切割对象
  -> 生成空间曲线刀路
  -> 编辑刀路工艺
  -> 规划刀路加工顺序
  -> 执行基于切割头机械模型的 CAM 仿真和碰撞检测
```

MVP 必须包含：

- 统一坐标系、单位和 `0.001 mm` 默认几何误差目标。
- 五轴切割头/刀具描述文件导入入口。
- 切割头子部件模型引用。
- 切割头关节链表达。
- 切割头 TCP、激光束方向和关键参数表达。
- STEP/STP、IGS/IGES 目标工件模型导入入口。
- 模型显示。
- 可拾取对象显示或选择。
- 自动识别候选切割对象。
- 手动拾取切割对象。
- 生成空间曲线刀路。
- 五轴姿态场。
- 刀路列表。
- 微联设置。
- 引线设置。
- 引出设置。
- 刀路顺序规划。
- 空移距离尽量减少的基础策略。
- 下刀点选择。
- 空移过程姿态调整。
- 蛙跳空移路径。
- 切割头机械模型仿真。
- 完整碰撞检测。
- 已切割刀路变红。
- 工程保存和再次打开。

MVP 不包含：

- 真实机床控制。
- NC 导出。
- 工件零点设置。
- 真实切割系统联机。
- 热变形模拟。
- 高级工艺库。
- 多机型适配。
- 云端排产或 MES 集成。

## 4. 核心概念

### 4.1 坐标系、单位与精度

本产品内部采用统一的三维右手直角坐标系表达几何、刀路、姿态和仿真状态。

基础约定：

- 长度单位为毫米。
- 角度单位为度。
- 时间单位为秒。
- 三维点、向量、曲线、姿态和包围盒都必须明确归属坐标系。
- 对外持久化数据和 UI 显示应使用上述单位。

坐标系定义：

- 项目坐标系 `ProjectCS` 是项目内所有 CAM 数据的公共基准坐标系。
- 模型坐标系 `ModelCS` 是目标工件 CAD 文件导入时携带或推导出的原始坐标系。
- 模型导入后应保存 `ModelCS -> ProjectCS` 的变换。
- 刀具装配坐标系 `ToolAssemblyCS` 是 SDF/SDFormat 中根装配导入后的基准坐标系。
- 子部件坐标系 `ComponentCS` 是切割头每个子部件自身的局部坐标系。
- 关节坐标系 `JointCS` 用于表达关节原点、关节轴和关节运动方向。
- TCP 坐标系 `TCPCS` 是切割头加工参考坐标系，TCP 坐标系 Z 轴约定为激光束方向。
- 刀路曲线数据默认存储在 `ProjectCS` 中。
- 五轴姿态场中的 TCP 位姿默认存储在 `ProjectCS` 中。
- 运动规划结果中的关节值使用刀具装配自身的关节定义表达，TCP 位姿仍以 `ProjectCS` 表达。

变换规则：

- `ModelCS -> ProjectCS` 用于把目标工件几何转换到项目空间。
- `ToolAssemblyCS -> ProjectCS` 用于把切割头装配放置到项目空间。
- `ComponentCS -> ParentComponentCS` 由 SDF/SDFormat 的 link、joint 和 pose 数据推导。
- 关节运动按照 `JointCS` 中定义的轴、原点、默认值和限位进行。
- TCP 最终位姿由切割头装配层级、关节值和 TCP 定义共同推导。
- 工件零点和真实机床坐标系不属于 MVP 范围，但不能影响项目内部坐标定义的完整性。

误差要求：

- 刀路曲线相对目标几何的默认最大几何偏差应控制在 `0.001 mm` 以内。
- 曲线拟合、候选对象转刀路、姿态场采样和运动规划都应以 `0.001 mm` 作为默认几何误差目标。
- 渲染网格可以使用较低精度的显示数据，但不能反向作为刀路和运动规划的几何依据。
- 碰撞检测中的安全距离不等同于几何误差，应使用刀具参数中的安全距离独立判断。

### 4.2 CAD 模型

CAD 模型是用户导入的三维几何来源。

要求：

- CAD 模型可以来自 STEP 等三维文件。
- MVP 目标工件模型支持 STEP/STP、IGS/IGES 两类模型文件导入。
- STL 不作为目标工件模型导入格式。
- STL 仅作为 SDF/SDFormat 切割头子部件模型或碰撞包络资源使用。
- CAD 模型用于显示、识别和拾取。
- CAD 模型不是 CAM 主数据。
- CAD 模型丢失后，已经确认的刀路仍应尽量可以显示、排序和仿真。

### 4.3 可拾取对象

可拾取对象是用户或算法从 CAD 模型中得到的候选输入。

典型类型：

- 边。
- loop。
- 后续扩展的外部导入曲线。

要求：

- 可拾取对象用于生成刀路。
- 可拾取对象可以保留来源信息。
- 可拾取对象不等于刀路。
- MVP 手动拾取的最小对象是 edge 和 loop。
- MVP 不支持手动画线生成刀路。
- 刀路生成后，后续加工流程不应强依赖可拾取对象继续存在。

### 4.4 切割刀路

切割刀路是 CAM 的核心数据。

要求：

- 刀路必须以三维空间曲线表达。
- 每条刀路必须有独立 ID。
- 每条刀路必须归属一个图层。
- 每条刀路必须可以独立显示、选择、排序、设置工艺和参与仿真。
- 用户可以手动设置刀路所属图层。
- 刀路可以记录来源信息，但来源信息是可选加分项，不是 MVP 必须项。
- 刀路不应以 CAD 模型拓扑引用作为主身份。

MVP 曲线能力：

- 直线。
- 圆弧。
- 折线。
- 椭圆。
- B 样条。
- 贝塞尔曲线。

B 样条要求：

- MVP 支持三阶 B 样条。

刀路持久化要求：

- 已确认的刀路必须持久化。
- 已确认的刀路曲线数据必须持久化。
- 已确认的刀路图层归属必须持久化。
- 项目重新打开后，已确认刀路不应依赖重新识别 CAD 拓扑才能恢复。

刀路图层要求：

- 系统应支持创建图层。
- 系统应支持重命名图层。
- 系统应支持为图层设置颜色。
- 系统应支持把一条或多条刀路手动分配到指定图层。
- 系统应支持按图层显示或隐藏刀路。
- 图层用于组织、显示、筛选和批量选择刀路，不改变刀路几何本身。

刀路规划质量要求：

- 刀路规划需要同时考虑切割精度效果和切割效率。
- 切割精度效果关注刀路是否贴合目标切割轮廓、是否满足允许误差、是否避免不必要的过切或欠切。
- 默认最大允许几何偏差为 `0.001 mm`。
- 切割效率关注刀路是否对后续运动规划友好。
- 运动规划友好性的核心指标是连续性。
- 位置连续是基本要求。
- 切向连续优于尖角折线。
- 曲率半径连续时，运动规划会更友好。
- 曲率半径变化曲线光滑时，运动规划会非常友好。
- 在不损害切割精度的前提下，系统应优先生成对运动规划更友好的曲线。

连续性目标层级：

```text
最低要求：位置连续
较好目标：切向连续
优秀目标：曲率半径连续
理想目标：曲率半径变化曲线光滑
```

### 4.4.1 五轴姿态场

五轴姿态场描述沿刀路曲线变化的切割头姿态，是五轴切割的必要数据。

要求：

- MVP 需要支持五轴姿态场。
- 姿态场以刀路曲线参数为自变量。
- 姿态场应能表达 TCP 位姿。
- 姿态场应能表达激光束方向。
- 姿态场应能表达或推导切割头关节值。
- 姿态场应能驱动切割头模型运动。
- 姿态场应能被仿真读取。

MVP 姿态场可以先采用离散采样表达，后续再扩展为连续函数或插值曲线。

### 4.4.2 五轴运动规划与安全检查

五轴运动规划把刀路曲线和五轴姿态场转换为切割头机械结构可执行的运动状态。

MVP 需要包含五轴运动规划和安全检查的核心链路，但不要求导出 NC 代码。

要求：

- 运动规划应读取刀具描述中的机械结构、关节类型、关节轴和运动范围。
- 运动规划应根据刀路曲线和五轴姿态场生成切割头关节运动序列。
- 运动规划应支持根据 TCP 位姿求解切割头关节值。
- 当同一个 TCP 位姿存在多个逆解时，应优先选择满足关节限位、远离奇异点、与上一运动状态变化最小的解。
- 运动规划应识别无逆解、逆解超限和逆解不连续的状态。
- 运动规划应识别奇异点或接近奇异点的状态。
- 当存在可行替代解时，系统应优先避开奇异点。
- 当无法避开奇异点时，安全检查结果应明确标记该问题。
- 运动规划应检查关节值是否超出刀具描述中的运动范围。
- 运动规划应检查关节运动是否存在不连续跳变。
- 运动规划应检查相邻采样点之间 TCP 位置、TCP 姿态和关节值是否连续。
- 闭合刀路或可反向刀路应支持选择下刀点和切割方向。
- 下刀点选择应在满足切割要求的前提下尽量减少抬刀、空移和姿态大幅变化。
- 空移过程中应允许切割头一边移动一边调整姿态，使到达下一条刀路下刀点时可以直接开始切割。
- 空移过程应支持蛙跳路径，即在必要时抬高到安全高度越过障碍，再下降到下一条刀路下刀点。
- 蛙跳路径也应参与关节限位、运动连续性和碰撞检查。
- 运动规划应输出可驱动切割头机械模型的运动状态。
- 安全检查应使用刀具描述中的运动范围、碰撞包络、工件模型和项目场景中存在的障碍物。
- 安全检查应包含关节限位检查、逆解检查、奇异点检查、运动连续性检查和碰撞检查。
- 真实切割系统中的工件零点、机床工件坐标系设置和联机不属于本产品 MVP 范围。

NC 代码输出可以后置。只要刀路曲线、五轴姿态场、运动规划和安全检查正确，NC 输出本质上是面向具体切割系统的翻译层。

### 4.5 刀路工艺

刀路工艺描述刀路如何切割。

MVP 工艺：

- 微联。
- 引线。
- 引出。

MVP 引线/引出能力：

- 直线。
- 圆弧。

MVP 微联要求：

- 微联位置采用曲线参数表达。

后续工艺：

- 激光功率。
- 切割速度。
- 气压。
- 焦点。
- 穿孔策略。
- 拐角策略。

要求：

- 工艺可以绑定到单条刀路。
- 工艺可以批量应用到多条刀路。
- 修改工艺后，预览和仿真应能反映变化。
- 工艺修改应支持撤销还原。

### 4.6 刀路顺序

刀路顺序描述多条刀路的加工先后关系。

要求：

- 刀路顺序应支持手动排序和自动排序两种大策略。
- 手动排序由用户直接调整刀路加工顺序。
- 自动排序由系统根据策略生成加工顺序。
- 系统可以计算空移。
- 自动排序应尽量减少空移距离。
- 系统可以决定某条刀路是否反向加工，用户也可以手动修改反向状态。
- 自动排序应结合下刀点选择、刀路反向状态和空移姿态调整共同计算。
- 自动排序可以生成普通空移段，也可以在必要时生成蛙跳空移段。

MVP 手动排序能力：

- 上移。
- 下移。
- 拖拽调整。
- 手动设置是否反向加工。

MVP 自动排序策略：

- 最近邻。
- 比较下一条刀路起点和终点，必要时反向。

### 4.7 CAM 仿真

CAM 仿真用于验证刀路顺序、切割头运动过程和碰撞风险。

MVP 仿真要求：

- 基于切割头机械模型进行仿真。
- 切割头模型应根据五轴姿态场或运动规划结果运动。
- 仿真播放速度 MVP 阶段采用固定速度。
- 空移过程可视化。
- 正在切割的刀路高亮。
- 已切割刀路显示为红色。
- 支持碰撞检测。
- 碰撞检测应覆盖切割头碰撞包络与工件模型之间的碰撞。
- 碰撞检测应覆盖切割头自身非相邻运动部件之间的自碰撞。
- 碰撞检测应覆盖切割头碰撞包络与项目场景中夹具、平台或其他静态障碍物之间的碰撞；当项目中没有这些对象时，该类检测自然为空。
- 碰撞检测应同时覆盖切割段、空移段和蛙跳段。
- 碰撞检测应区分安全距离不足和实际相交两类结果。
- 碰撞检测结果应能在视图中反馈。
- 可以开始、暂停、停止和重置仿真。

MVP 仿真不要求：

- 真实机床加减速。
- 真实材料去除。

### 4.8 五轴切割头/刀具系统

五轴切割头/刀具系统描述激光切割执行端的装配、运动链、TCP、激光束方向、子部件模型和必要参数。

这里的“刀具”不是传统铣刀意义上的单一几何体，而是一套可运动的切割头装配：

```text
ToolAssembly
  -> ToolComponent[]
  -> Joint[]
  -> TCP
  -> BeamAxis
  -> Parameters
  -> CollisionGeometry[]
```

要求：

- 系统应支持导入一个 SDF/SDFormat 刀具描述文件。
- 刀具描述文件应能引用多个子部件模型文件。
- 子部件模型文件 MVP 支持 STEP/STP 和 STL。
- 刀具描述文件应能表达子部件之间的父子关系。
- 刀具描述文件应能表达关节类型、关节轴、零位、限位和默认值。
- 刀具描述文件应能表达 TCP。
- 刀具描述文件应能表达激光束方向。
- 刀具描述文件应能表达喷嘴、焦点、保护距离等必要参数。
- 刀具描述文件应能表达用于显示的模型和用于碰撞/避让的简化包络。
- 后续刀路规划和运动规划应能使用这些参数。
- 后续仿真应能根据运动规划驱动刀具模型运动。

MVP 关节能力：

- 固定关节。
- 旋转关节。
- 平移关节。

MVP 参数：

- ToolID。
- ToolName。
- TCP 位置。
- TCP 坐标系。
- TCP 坐标系 Z 轴约定为激光束方向。
- 喷嘴长度。
- 喷嘴半径。
- 安全距离。
- 最大摆角。
- 关节限位。

刀具描述格式：

- 产品原生刀具描述文件采用 SDF/SDFormat。
- SDF 的 model/link/joint/pose 结构用于表达切割头机械结构、子部件、父子关系、关节和位姿。
- SDF 的 visual/collision 结构用于表达子部件显示模型和碰撞/避让包络。
- 激光切割专属字段，如 TCP、激光束方向、喷嘴、焦点、安全距离、最大摆角等，通过 iCAX 扩展字段表达。
- TCP 坐标系约定 Z 轴为激光束方向。
- 碰撞/避让包络 MVP 支持 mesh 和包围盒。
- 碰撞/避让包络 MVP 不支持胶囊体。
- SDF 文件导入后应转换为产品内部统一的 `ToolAssembly` 运行时数据结构。
- 项目保存时应保存可复原的刀具描述数据，不能依赖临时运行时句柄。

外部数据标准参考：

- SDF/SDFormat：作为产品原生刀具描述文件格式。
- ISO 13399：切削刀具数据表示与交换，适合参考刀具分类、属性和交换思想。
- MTConnect Cutting Tools：适合参考设备侧刀具资产表达和运行数据交换。
- STEP-NC/AP238：适合参考制造过程、刀路和设备无关制造数据交换。

格式选择不能牺牲机械结构表达能力。准确表达父子结构、关节类型、关节轴、行程/角度限制、TCP、激光束方向和必要参数是第一优先级。

## 5. 用户工作流规格

### 5.1 新建项目

用户可以创建一个三维线条切割 CAM 项目。

验收：

- 项目具有产品标识。
- 项目可以保存。
- 项目可以再次打开。

### 5.2 导入五轴切割头/刀具描述文件

用户可以导入一个 SDF/SDFormat 五轴切割头/刀具描述文件。

验收：

- 导入后系统能显示刀具名称。
- 系统能加载 STEP/STP 或 STL 子部件模型。
- 系统能显示刀具装配结构。
- 系统能识别 TCP。
- 系统能识别激光束方向。
- 系统能识别关节类型、关节轴和关节限位。
- 系统能在视图区显示切割头模型。
- 系统能根据给定关节值驱动刀具模型运动。
- 描述文件缺少必要字段时，系统应给出明确错误。

### 5.3 导入 CAD 模型

用户可以选择一个三维 CAD 文件导入项目。

验收：

- 用户可以导入 STEP/STP 目标工件模型。
- 用户可以导入 IGS/IGES 目标工件模型。
- 目标工件模型不支持 STL 导入。
- 导入后模型出现在视图区。
- UI 显示模型名称或路径。
- 项目中记录模型来源。
- 项目中保存导入后的目标工件资源，不能只依赖原始外部文件路径。
- 大模型数据不通过 Mailbox 传输。

### 5.4 自动识别候选切割对象

系统可以根据 CAD 模型识别候选切割对象。

验收：

- MVP 支持孔特征识别。
- MVP 支持坡口孔识别，包括 V 坡口和 X 坡口等类型。
- 识别结果出现在候选列表或视图中。
- 候选对象可以被选中。
- 候选对象可以生成刀路。
- 识别失败时，用户仍可手动拾取。

### 5.5 手动拾取切割对象

用户可以在视图中拾取切割对象。

验收：

- 鼠标悬停时可以反馈候选对象。
- 点击后产生当前选择。
- 当前选择可以加入待生成列表。
- 用户可以取消选择或清空列表。

### 5.6 生成空间曲线刀路

用户可以基于候选对象或手动拾取对象生成刀路。

验收：

- 每条生成结果都有独立刀路 ID。
- 每条刀路有空间曲线数据。
- 刀路可以显示在视图区。
- 刀路可以出现在刀路列表中。
- 刀路生成后不依赖原 CAD 拓扑作为主身份。
- 刀路生成结果应能给出切割精度相关摘要，如拟合误差或偏差状态。
- 刀路生成结果的默认最大几何偏差目标为 `0.001 mm`。
- 刀路生成结果应能给出运动友好性相关摘要，如连续性等级。
- 在切割精度允许范围内，生成结果应优先选择连续性更好的曲线。

### 5.7 编辑刀路工艺

用户可以给刀路设置微联、引线和引出。

验收：

- 用户可以选择一条或多条刀路。
- 用户可以设置微联数量、位置和长度。
- 用户可以设置引线类型和参数。
- 用户可以设置引出类型和参数。
- 修改后视图能展示工艺效果或摘要。

### 5.8 设置刀路图层

用户可以手动设置一条或多条刀路所属图层。

验收：

- 用户可以创建图层。
- 用户可以重命名图层。
- 用户可以设置图层颜色。
- 用户可以把选中的刀路移动到指定图层。
- 刀路列表能显示刀路所属图层。
- 视图能按图层颜色显示刀路。
- 用户可以按图层显示或隐藏刀路。

### 5.9 生成五轴运动规划并执行安全检查

用户可以基于刀路曲线、五轴姿态场和刀具机械表达生成运动规划，并执行安全检查。

验收：

- 系统能生成切割头关节运动序列。
- 系统能根据 TCP 位姿求解切割头关节值。
- 系统能在存在多个逆解时选择更连续、更远离限位和奇异点的解。
- 系统能识别无逆解、逆解超限、逆解不连续和奇异点风险。
- 系统能为闭合刀路或可反向刀路选择下刀点和切割方向。
- 下刀点选择结果应尽量减少抬刀、空移和姿态大幅变化。
- 空移过程中切割头姿态应逐步调整，到达下一条刀路下刀点时具备直接开始切割的姿态。
- 系统支持蛙跳空移路径。
- 蛙跳路径应参与关节限位、连续性和碰撞检查。
- 系统能检查关节运动范围。
- 系统能检查运动连续性。
- 系统能执行切割头与工件模型之间的碰撞检测。
- 系统能执行切割头自身非相邻运动部件之间的自碰撞检测。
- 系统能执行切割头与项目场景静态障碍物之间的碰撞检测。
- 系统能区分安全距离不足和实际相交。
- 系统能显示安全检查结果。
- 安全检查失败时，系统能指出失败原因。

### 5.10 规划刀路顺序

用户可以通过手动排序或自动排序得到加工顺序。

验收：

- 顺序列表包含所有参与加工的刀路。
- 用户可以手动调整刀路顺序。
- 用户可以手动设置刀路是否反向。
- 系统可以自动生成顺序列表。
- 自动排序后系统给出空移长度。
- 自动排序可以标记刀路是否反向。
- 自动排序可以给出下刀点选择结果。
- 自动排序可以生成空移段和蛙跳段。
- 视图中可以展示加工顺序。

### 5.11 CAM 机械模型仿真

用户可以执行基于切割头机械模型的加工仿真。

验收：

- 点击开始后切割头机械模型沿规划顺序运动。
- 切割头机械模型的关节状态随仿真进度变化。
- 当前切割刀路高亮。
- 已切割完成的刀路变为红色。
- 空移段、蛙跳段和切割段可以区分。
- 发生碰撞时，视图中应能看到碰撞提示。
- 用户可以暂停、继续、停止和重置。

## 6. 数据规格

### 6.1 项目数据

项目应保存：

- 产品 ID。
- 当前刀具描述。
- 模型来源。
- 内嵌目标工件模型资源。
- 内嵌刀具描述文件资源。
- 内嵌刀具子部件模型资源。
- 内嵌碰撞包络资源。
- 已确认刀路。
- 已确认刀路曲线。
- 刀路图层。
- 五轴姿态场。
- 五轴运动规划结果。
- 安全检查结果。
- 刀路工艺。
- 刀路顺序。
- 用户确认的必要参数。

项目可以保存：

- 可拾取对象索引。
- 特征识别结果。
- 显示缓存。
- 仿真缓存。

项目不应保存：

- 前端临时 hover 状态。
- 几何内核对象指针。
- 不可复原的运行时句柄。

项目资源规则：

- 资源必须跟随项目文件保存，不能只保存外部路径。
- 原始外部路径可以作为来源元数据保存，但不能作为恢复项目的唯一依据。
- 目标工件 STEP/STP、IGS/IGES 文件导入后应作为项目内嵌资源保存。
- SDF/SDFormat 刀具描述文件导入后应作为项目内嵌资源保存。
- SDF/SDFormat 引用的 STEP/STP 或 STL 子部件模型应作为项目内嵌资源保存。
- 碰撞包络资源应作为项目内嵌资源保存。
- 重新打开项目时，不要求原始外部文件仍然存在。

### 6.2 五轴切割头/刀具数据

刀具描述应包含：

- ToolID。
- ToolName。
- 子部件列表。
- 关节列表。
- TCP。
- 激光束方向。
- 关键参数。
- 碰撞/避让包络。

每个子部件应包含：

- ComponentID。
- Name。
- ParentComponentID。
- ModelResourceID。
- LocalTransform。
- 可选 CollisionResourceID。

每个关节应包含：

- JointID。
- ParentComponentID。
- ChildComponentID。
- JointType。
- Axis。
- Origin。
- DefaultValue。
- MinValue。
- MaxValue。

JointType MVP 支持：

- Fixed。
- Revolute。
- Prismatic。

TCP 应包含：

- 所属子部件。
- TCP 原点。
- TCP 坐标系。
- 激光束方向。

刀具参数应至少包含：

- 喷嘴长度。
- 喷嘴半径。
- 焦点偏移。
- 安全距离。
- 最大摆角。

### 6.3 刀路数据

每条刀路应包含：

- 刀路 ID。
- 名称。
- 图层 ID。
- 空间曲线资源 ID。
- 刀路质量摘要。
- 工艺引用或工艺覆盖。
- 默认下刀点参数。
- 默认切割方向。
- 状态。
- 版本。
- 可选来源信息。

刀路曲线应包含：

- 曲线 ID。
- 是否闭合。
- 曲线段集合。
- 曲线版本。

曲线段应至少支持：

- 直线段。
- 圆弧段。
- 折线段。
- 椭圆段。
- 三阶 B 样条段。
- 贝塞尔段。

刀路质量摘要应包含：

- 切割精度状态。
- 可选最大偏差，单位为毫米。
- 连续性等级。
- 可选最小曲率半径。
- 可选曲率半径变化平滑性摘要。

### 6.4 图层数据

每个图层应包含：

- LayerID。
- Name。
- Color。
- Visible。
- LockState。

图层规则：

- 每条刀路必须属于一个图层。
- 系统应提供默认图层。
- 删除图层前必须处理该图层下的刀路归属。
- 图层显示隐藏只影响视图表现，不删除刀路数据。

### 6.5 五轴姿态场数据

五轴姿态场应包含：

- 姿态场 ID。
- 关联刀路 ID。
- 姿态场版本。
- 采样点列表。

每个姿态采样点应至少包含：

- 曲线参数。
- TCP 位置。
- TCP 姿态。
- 激光束方向。
- 可选关节值。

MVP 阶段允许姿态场采用离散采样点表达。后续可以扩展为连续姿态曲线或插值规则。

### 6.6 五轴运动规划数据

五轴运动规划结果应包含：

- 运动规划 ID。
- 关联刀路 ID。
- 运动规划版本。
- 下刀点选择结果。
- 关节运动序列。
- 空移段列表。
- 蛙跳段列表。
- 逆解检查结果。
- 奇异点检查结果。
- 关节限位检查结果。
- 运动连续性检查结果。
- 碰撞检查结果。

每个关节运动采样点应至少包含：

- 曲线参数。
- 运动段 ID。
- TCP 位置。
- TCP 姿态。
- 关节值列表。
- 逆解状态。
- 是否接近奇异点。
- 是否处于切割状态。
- 是否处于空移状态。
- 是否处于蛙跳状态。

安全检查结果应包含：

- 是否通过。
- 关节限位违规列表。
- 逆解失败列表。
- 奇异点风险列表。
- 碰撞事件列表。
- 运动不连续事件列表。

每个碰撞事件应至少包含：

- 发生位置或采样点。
- 所属刀路 ID。
- 所属运动段类型，包含切割段、空移段或蛙跳段。
- 碰撞对象 A。
- 碰撞对象 B。
- 结果类型，包含安全距离不足或实际相交。
- 可选碰撞点或最近点。
- 可选穿透深度或最小距离。

### 6.7 工艺数据

每条刀路的 MVP 工艺应支持：

- 微联列表。
- 引线设置。
- 引出设置。

每个微联应至少包含：

- 位置，采用曲线参数表达。
- 长度。

每个引线或引出应至少包含：

- 类型，MVP 支持直线和圆弧。
- 长度。
- 半径。
- 角度。

### 6.8 顺序数据

顺序计划应包含：

- 计划版本。
- 刀路 ID 列表。
- 每条刀路是否反向。
- 估算空移总长度。

### 6.9 仿真数据

仿真状态应包含：

- 是否播放中。
- 当前段。
- 当前参数。
- 当前 TCP 位置。
- 当前切割头关节值。
- 当前切割头位姿。
- 固定仿真速度。
- 当前碰撞状态。
- 当前碰撞对象。
- 已完成刀路 ID 列表。

仿真状态可以是运行时数据，不要求持久化。

### 6.10 资源数据

资源是项目文件中可复原的数据载体。

每个资源应至少包含：

- ResourceID。
- ResourceType。
- OriginalPath。
- ContentHash。
- BinaryContent。
- Metadata。

MVP 资源类型应至少包含：

- 目标工件 CAD 资源，支持 STEP/STP、IGS/IGES。
- 刀具描述资源，支持 SDF/SDFormat。
- 刀具子部件模型资源，支持 STEP/STP、STL。
- 碰撞包络资源，支持 mesh 和包围盒。

资源保存规则：

- 项目保存时必须把 MVP 工作流可复原所需资源内嵌到项目文件。
- `OriginalPath` 仅用于用户查看来源和后续重新关联，不参与项目恢复的必要链路。
- 显示网格可以作为缓存资源保存，也可以重新生成；它不是刀路和运动规划的权威数据。

## 7. UI 规格

MVP UI 应按 CAM 工作台组织，而不是按临时功能按钮堆叠。

推荐主布局：

- 顶部工具栏：项目、导入刀具、导入模型、识别、生成刀路、生成运动规划、安全检查、仿真。
- 左侧流程区：按“刀具、模型、识别/拾取、刀路、图层、顺序、仿真”组织工作流。
- 中央三维视图区：承载模型、切割头、候选对象、刀路、姿态、空移、蛙跳、碰撞和仿真状态。
- 右侧属性区：显示当前选中对象的属性和可编辑参数。
- 底部数据区：显示刀路列表、候选对象列表、碰撞/安全检查列表和仿真时间状态。

顶部工具栏应支持：

- 新建、打开、保存项目。
- 导入 SDF/SDFormat 切割头描述文件。
- 导入 STEP/STP、IGS/IGES 目标工件模型。
- 执行特征识别。
- 基于选择生成刀路。
- 生成五轴姿态场和运动规划。
- 执行安全检查。
- 启动、暂停、继续、停止、重置仿真。

左侧流程区应支持：

- 查看当前项目状态。
- 查看当前切割头是否已导入。
- 查看当前目标工件是否已导入。
- 切换拾取过滤器，MVP 至少支持 edge 和 loop。
- 查看识别得到的孔和坡口孔候选对象。
- 查看图层树。
- 查看加工顺序。
- 查看仿真状态。

中央三维视图区应支持：

- 显示目标工件模型。
- 显示切割头装配模型。
- 显示切割头碰撞包络。
- 显示候选 edge 和 loop。
- 显示当前鼠标悬停对象。
- 显示当前选择对象。
- 显示已确认刀路。
- 按图层颜色显示刀路。
- 显示刀路方向和下刀点。
- 显示引线、引出和微联。
- 显示五轴姿态场。
- 显示空移路径。
- 显示蛙跳路径。
- 显示运动规划后的切割头姿态。
- 显示关节限位违规位置。
- 显示奇异点风险位置。
- 显示运动不连续事件位置。
- 显示安全距离不足位置。
- 显示实际碰撞位置。
- 仿真时显示正在切割的刀路。
- 仿真时把已切割刀路显示为红色。

右侧属性区应根据当前选择切换内容：

- 选中切割头时，显示 ToolID、ToolName、TCP、关节、限位、安全距离和最大摆角。
- 选中目标工件时，显示模型来源、模型坐标变换和资源状态。
- 选中候选对象时，显示候选类型、来源和可生成刀路状态。
- 选中刀路时，显示刀路名称、图层、状态、连续性等级、最大偏差、工艺和版本。
- 选中图层时，显示图层名称、颜色、显示状态和锁定状态。
- 选中碰撞或安全检查事件时，显示事件类型、发生位置、关联刀路、关联运动段和相关对象。

底部刀路列表应支持：

- 显示刀路名称。
- 显示刀路状态。
- 显示刀路图层。
- 显示刀路加工顺序。
- 显示是否反向加工。
- 显示下刀点状态。
- 显示最大偏差。
- 显示连续性等级。
- 显示安全检查状态。
- 选择刀路。
- 批量选择刀路。
- 手动拖拽调整加工顺序。
- 手动设置是否反向加工。
- 把选中刀路移动到指定图层。

图层管理应支持：

- 创建图层。
- 重命名图层。
- 设置图层颜色。
- 显示或隐藏图层。
- 锁定或解锁图层。
- 把选中刀路移动到指定图层。
- 按图层筛选刀路列表。

安全检查列表应支持：

- 显示关节限位违规。
- 显示逆解失败。
- 显示奇异点风险。
- 显示运动不连续事件。
- 显示安全距离不足事件。
- 显示实际碰撞事件。
- 点击事件后在三维视图区定位到对应位置。

仿真控制应支持：

- 开始。
- 暂停。
- 继续。
- 停止。
- 重置。
- 显示当前刀路。
- 显示当前段类型，包含切割段、空移段或蛙跳段。
- 显示当前 TCP 位置。
- 显示当前关节值。
- 显示当前碰撞状态。

## 8. 通信规格

Mailbox 用于：

- 导入刀具描述文件命令。
- 导入模型命令。
- 识别命令。
- 拾取结果提交。
- 生成刀路命令。
- 选择或更新下刀点命令。
- 生成五轴运动规划命令。
- 执行安全检查命令。
- 修改工艺命令。
- 修改图层命令。
- 规划顺序命令。
- 仿真控制命令。

PDO 用于：

- 刀具运动显示状态。
- 大模型显示数据。
- 刀路显示数据。
- 五轴姿态场显示数据。
- 五轴运动规划显示数据。
- 空移路径显示数据。
- 蛙跳路径显示数据。
- 安全检查结果显示数据。
- 仿真切割头状态。
- 仿真碰撞检测状态。
- 已切割状态。
- 高频输入状态。

Mailbox 不用于：

- 大规模 mesh。
- 大规模刀路采样点。
- 每帧高频状态。

## 9. 撤销还原规格

应支持撤销还原的操作：

- 导入或切换刀具描述。
- 导入模型。
- 生成刀路。
- 删除刀路。
- 生成或更新五轴运动规划。
- 修改刀路工艺。
- 修改刀路图层。
- 创建、重命名和修改图层。
- 修改刀路顺序。
- 手动调整刀路。

不要求支持撤销还原的操作：

- 鼠标 hover。
- 仿真播放位置。
- 临时视图状态。

## 10. 验收标准

MVP 验收以一条完整工作流为准：

1. 新建项目。
2. 导入五轴切割头/刀具描述文件。
3. 显示切割头模型。
4. 修改或给定关节值后，切割头模型能运动。
5. 导入 STEP/STP 或 IGS/IGES 目标工件模型。
6. 显示模型。
7. 自动识别孔或坡口孔，或手动拾取 edge/loop。
8. 生成至少一条空间曲线刀路。
9. 刀路能显示切割精度摘要，默认最大几何偏差目标为 `0.001 mm`。
10. 生成该刀路对应的五轴姿态场。
11. 基于五轴姿态场执行逆解，生成切割头关节运动序列。
12. 系统能识别无逆解、超限、奇异点和不连续问题。
13. 系统能选择下刀点，并在空移中逐步调整姿态。
14. 系统能生成蛙跳空移路径。
15. 执行关节运动范围检查、运动连续性检查和完整碰撞检查。
16. 碰撞检查能覆盖切割头与工件、切割头自碰撞、切割头与项目静态障碍物。
17. 给刀路添加直线/圆弧引线、引出或基于曲线参数的微联。
18. 手动设置该刀路所属图层。
19. 生成刀路顺序。
20. 启动固定速度切割头机械模型仿真。
21. 切割头模型沿刀路、空移段或蛙跳段运动，关节状态能按运动状态联动。
22. 仿真过程中能输出并显示碰撞检测状态。
23. 已切割刀路变红。
24. 保存项目。
25. 重新打开项目后，内嵌资源、刀具描述、已确认刀路、刀路曲线、刀路图层、五轴姿态场、五轴运动规划结果、工艺和顺序仍然存在。
