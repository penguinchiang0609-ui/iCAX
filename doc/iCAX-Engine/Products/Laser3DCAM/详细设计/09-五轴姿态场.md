# 09 五轴姿态场详细设计

## 1. 模块定位

五轴姿态场描述沿刀路曲线变化的切割头 TCP 位姿和激光束方向，是刀路到机械运动之间的桥。

MVP 使用离散采样表达。

## 2. 输入和输出

输入：

- ToolpathCurveResource。
- ToolAssembly。
- 用户或算法提供的姿态规则。
- 项目坐标系和误差设置。

输出：

- PoseFieldResource。
- 姿态场显示数据。

## 3. 数据结构

```text
PoseFieldResource
  PoseFieldID
  ToolpathID
  Version
  Sample[]
```

```text
PoseSample
  CurveParameter
  TCPPositionInProjectCS
  TCPPoseInProjectCS
  BeamDirectionInProjectCS
  OptionalJointValues
```

## 4. 采样策略

MVP 采用自适应采样：

- 曲率越大，采样越密。
- 姿态变化越大，采样越密。
- 采样几何误差目标为 `0.001 mm`。
- 起点和终点必须有采样点。
- 闭合刀路下刀点必须有采样点。

## 5. 姿态生成原则

基础原则：

- TCP 位置位于刀路曲线上。
- TCP Z 轴为激光束方向。
- 激光束方向应满足切割要求。
- 姿态变化应尽量平滑。

坡口孔：

- 激光束方向需要表达坡口角。
- V 坡口和 X 坡口可产生不同姿态规则。

## 6. 姿态插值

MVP 内部可使用离散采样。

后续可扩展：

- TCP 位置连续曲线。
- 姿态四元数插值。
- 关节空间插值。

当前持久化只要求样本列表可复原。

## 7. 与逆解关系

PoseField 可以包含 OptionalJointValues，但不是必须。

```text
PoseField -> KinematicsService -> JointValues
```

若 OptionalJointValues 存在，KinematicsService 仍需校验：

- 关节限位。
- TCP 位姿误差。
- 连续性。
- 奇异点。

## 8. 生成流程

```text
1. 读取 ToolpathCurveResource
2. 根据曲线和工艺确定有效切割区间
3. 生成采样参数
4. 计算 TCP 位置
5. 计算激光束方向
6. 构造 TCP 姿态
7. 写入 PoseFieldResource
8. 通知渲染同步
```

## 9. 失败处理

- 刀路曲线不存在：失败。
- 曲线参数无效：失败。
- 无法生成合法激光束方向：失败。
- 姿态突变过大：标记警告或失败。

## 10. 测试点

- 每条刀路能生成 PoseFieldResource。
- 采样点 TCP 位于刀路曲线上。
- BeamDirection 与 TCPCS Z 轴一致。
- 闭合刀路下刀点有采样点。
- 保存重开后姿态场可恢复。
