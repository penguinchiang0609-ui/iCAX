# 09 五轴姿态场详细设计

## 1. 模块定位

五轴姿态场描述沿刀路曲线变化的世界坐标姿态和激光束方向，是刀路到机械运动之间的桥。

MVP 使用离散采样表达。

## 2. 输入和输出

输入：

- PathCurveResource。
- ToolAssembly。
- 用户或算法提供的姿态规则。
- 世界坐标和误差设置。

输出：

- PoseFieldResource。
- 姿态场显示数据。

## 3. 数据结构

```text
PoseFieldResource
  Version
  Sample[]
```

```text
PoseSample
  SegmentIndex
  SegmentU
  BeamDirection
```

PoseFieldResource 是纯数据资源，不反向保存 ToolpathID、CurveResourceID、CurveVersion 或 PoseFieldID。资源身份由 ResourcePool key 表达；刀路到姿态场的归属关系由 `CCAMPathComponent.PoseFieldResourceID` 表达；曲线和姿态场之间的对应关系由同一个 `CCAMPathComponent` 同时持有 `CurveResourceID` 和 `PoseFieldResourceID` 来维护。

采样位置通过 `SegmentIndex + SegmentU` 表达：`SegmentIndex` 指向 `PathCurveResource.Curve.Segments[]` 中的某一段，`SegmentU` 是该段曲线自己的原生参数。姿态场不使用整条刀路归一化参数，也不使用累计弧长作为主定位。`BeamDirection` 使用世界坐标。采样点位置不持久化在资源中，而是由曲线段按 `SegmentU` 求值得到。PoseFieldResource 不引入 TCP 字段、完整机械姿态或关节值。TCP 是切割头机械模型的概念，由运动规划和逆解阶段根据切割头定义解释。

## 4. 采样策略

MVP 采用自适应采样：

- 曲率越大，采样越密。
- 姿态变化越大，采样越密。
- 采样几何误差目标为 `0.001 mm`。
- 起点和终点必须有采样点。
- 闭合刀路下刀点必须有采样点。
- 对复合曲线按段采样；每个采样点保存所在段的 `SegmentU`。

## 5. 姿态生成原则

基础原则：

- 采样位置位于刀路曲线上。
- 采样姿态中的激光束方向使用世界坐标向量表达。
- 激光束方向应满足切割要求。
- 姿态变化应尽量平滑。

坡口孔：

- 激光束方向需要表达坡口角。
- V 坡口和 X 坡口可产生不同姿态规则。

## 6. 姿态插值

MVP 内部可使用离散采样。

后续可扩展：

- 位置连续曲线。
- 姿态四元数插值。

当前持久化只要求样本列表可复原。

## 7. 与逆解关系

PoseField 不保存关节值。关节值由 KinematicsService 或 MotionPlanningService 根据姿态场、切割头机械模型和约束求解。

```text
PoseField -> KinematicsService -> JointValues
```

KinematicsService 需要校验：

- 关节限位。
- 世界坐标姿态误差。
- 连续性。
- 奇异点。

## 8. 生成流程

```text
1. 读取 PathCurveResource
2. 根据曲线和工艺确定有效切割区间
3. 按曲线段生成 SegmentIndex 和 SegmentU
4. 计算世界坐标位置
5. 计算激光束方向
6. 写入 PoseFieldResource
7. 通知渲染同步
```

## 9. 失败处理

- 刀路曲线不存在：失败。
- 曲线参数无效：失败。
- 无法生成合法激光束方向：失败。
- 姿态突变过大：标记警告或失败。

## 10. 测试点

- 每条刀路能生成 PoseFieldResource。
- 采样点位置位于刀路曲线上。
- 采样点能通过 SegmentIndex + SegmentU 回到对应曲线段。
- BeamDirection 使用世界坐标表达，且与姿态定义一致。
- 闭合刀路下刀点有采样点。
- 保存重开后姿态场可恢复。
