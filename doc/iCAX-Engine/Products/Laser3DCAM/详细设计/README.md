文档分层：

- 规格文档：定义产品要做什么。
- 方案文档：定义产品如何在 iCAX 框架下落地。
- 详细设计：定义每个模块如何开发、如何协作、如何测试。

统一约束：

- 产品逻辑不得写入 `iCAX-Engine` framework。
- 项目主数据进入 Project.Database。
- 大资源进入 Project.Resources，并随项目文件内嵌。
- Mailbox 只传命令和小型 JSON。
- PDO 承担高频和大块数据同步。
- 目标工件只支持 STEP/STP、IGS/IGES。
- STL 只用于切割头子部件模型或碰撞包络资源。
- 默认几何误差目标为 `0.001 mm`。
- 坐标系不作为独立功能模块展开；除切割头机械结构内部数据外，产品业务数据默认使用世界坐标。
