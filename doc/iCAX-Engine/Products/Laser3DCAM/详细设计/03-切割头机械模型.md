# 03 切割头机械模型详细设计

## 1. 模块定位

切割头机械模型用于表达五轴激光切割执行端的装配结构、关节链、TCP、激光束方向、显示模型、碰撞包络和关键参数。

它是后续姿态驱动、逆解、运动规划、碰撞检测和仿真的共同基础。

## 2. 输入和输出

输入：

- SDF/SDFormat 描述文件。
- 子部件 STEP/STP 或 STL 模型。
- 碰撞 mesh 或包围盒。
- iCAX 扩展字段。

输出：

- `ToolAssemblyResource`。
- `CLaserCamToolAssemblyComponent`。
- 切割头显示数据。
- 切割头碰撞数据。

## 4. 关节类型

Fixed：

- 无自由度。
- 子部件相对父部件固定。

Revolute：

- 绕 `AxisInJointCS` 旋转。
- 关节值单位为度。
- 使用 MinValue/MaxValue 表达角度限位。

Prismatic：

- 沿 `AxisInJointCS` 平移。
- 关节值单位为毫米。
- 使用 MinValue/MaxValue 表达行程限位。

## 5. 正向运动学

目标：给定关节值，计算每个 Component 和 TCP 在 ProjectCS 下的位姿。

流程：

```text
1. 读取 ToolAssemblyToProjectTransform
2. 从 RootComponent 开始遍历装配树
3. 对每个 Joint 计算 JointMotionTransform
4. ComponentWorld = ParentWorld * JointOrigin * JointMotion * ComponentLocal
5. TCPWorld = ComponentWorld * TCPTransform
```

伪代码：

```text
ComputeWorldPose(component):
  if component is root:
    return ToolAssemblyToProject

  parentPose = ComputeWorldPose(parent)
  joint = Joint(parent, component)
  jointPose = joint.OriginInParentCS
  motion = BuildJointMotion(joint.Type, joint.Axis, joint.Value)
  return parentPose * jointPose * motion * component.LocalTransform
```

## 6. 驱动运动

驱动源：

- 用户手动给定关节值。
- PoseField 中的 OptionalJointValues。
- MotionPlanResource 中的 MotionSample.JointValues。

驱动过程：

```text
MotionSample
  -> JointValueSet
  -> ToolAssemblyService.ApplyJointValues()
  -> ForwardKinematics()
  -> ComponentPose[]
  -> RenderPDO / SimulationState / CollisionScene
```

## 7. 碰撞包络

每个 ToolComponent 可有碰撞包络：

- mesh。
- 包围盒。

规则：

- 显示模型和碰撞模型可以不同。
- 碰撞包络必须跟随组件运动。
- 自碰撞检测时，相邻父子组件默认可以配置为忽略，非相邻运动部件必须检测。

## 8. 数据生命周期

导入时：

- 生成 ToolAssemblyResource。
- 生成 ToolAssemblyComponent。
- 子部件模型和碰撞资源内嵌到项目。

运行时：

- ToolAssemblyService 加载 ToolAssemblyResource。
- 缓存装配树和关节索引。
- 每帧根据 MotionPlan 更新关节值。

保存时：

- 保存 SDF 原始资源。
- 保存解析后的 ToolAssemblyResource。
- 保存子部件资源和碰撞资源。

## 9. 失败处理

- 装配树有环：导入失败。
- 子部件引用不存在：导入失败。
- Joint parent/child 不存在：导入失败。
- TCP 未定义：导入失败。
- 关节限位无效：导入失败。
- BeamAxis 与 TCP 约定冲突：导入失败。

## 10. 测试点

- Fixed/Revolute/Prismatic 正向运动学结果正确。
- TCP Z 轴与激光束方向一致。
- 子部件模型随关节运动正确联动。
- 碰撞包络随组件运动。
- 保存重开后 ToolAssembly 可完全复原。
