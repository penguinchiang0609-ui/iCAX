# 08 图层工艺与顺序规划详细设计

## 1. 模块定位

本模块负责刀路组织、MVP 工艺参数和加工顺序规划。

MVP 工艺仅包含：

- 微联。
- 引线。
- 引出。

激光功率、切割速度、气体、焦点、穿孔策略暂不进入当前模块。

## 2. CuttingLayer 与 VisibleLayer

CAM 中的图层分为两类，不能混用：

- `CuttingLayer`：给切割系统使用，用于加工工艺分组。
- `VisibleLayer`：给前端使用，用于显示、隐藏、颜色和选择过滤。

```text
CLaserCamCuttingLayerComponent
  Name
  CuttingProcessID
  CuttingProcessName
  Enabled
  OutputOrder
```

```text
CLaserCamVisibleLayerComponent
  Name
  Color
  Visible
  Locked
  Order
```

规则：

- 每条刀路必须同时属于一个 CuttingLayer 和一个 VisibleLayer。
- 项目创建时自动创建默认 CuttingLayer 和默认 VisibleLayer。
- CuttingLayer 决定切割系统输出时的工艺分组，不负责颜色和显隐。
- VisibleLayer 隐藏只影响显示，不删除刀路，也不影响切割输出。
- VisibleLayer 锁定后不允许通过前端交互移动或修改该层下刀路。
- CuttingLayer 的 `CuttingProcessID` 是切割系统可识别的工艺 ID；iCAX 当前不展开功率、速度、气体、焦点等具体参数。

## 3. 工艺数据

```text
ToolpathProcess
  MicroJoint[]
  LeadIn
  LeadOut
```

```text
MicroJoint
  CurveParameter
  Length
```

```text
LeadInOut
  Type(Line/Arc)
  Length
  Radius
  Angle
```

曲线参数使用 `[0, 1]`。

## 4. 工艺修改流程

```text
1. 前端选择刀路
2. 用户修改微联/引线/引出
3. Cam.UpdateToolpathProcess
4. ModifyFilter 检查参数合法性
5. Repository 修改 ToolpathComponent
6. Observer 通知渲染和属性面板刷新
```

## 5. 工艺校验

微联：

- 参数必须在 `[0, 1]`。
- 长度必须大于 0。
- 多个微联不能重叠。

引线/引出：

- 类型必须是 Line 或 Arc。
- 长度必须大于 0。
- Arc 半径必须大于 0。

## 6. 顺序数据

```text
CCAMBlockComponent
  Children[]
```

```text
ProgramChild
  Kind(block/path)
  EntityID
```

顺序规划的权威结果写在 Block 的 `Children[]` 中。一个加工文件是根 Block；用户可以把若干刀路组成子 Block，也可以把子 Block 继续组合成更大的 Block。

需要保存方向和下刀点时，写入 Toolpath 自身或后续独立的工艺/姿态数据：

```text
ToolpathOrderHint
  Direction(Forward/Reverse)
  LeadInParameter
```

## 7. 手动排序

支持：

- 上移。
- 下移。
- 拖拽。
- 手动正反向。
- 手动设置下刀点。

手动排序直接修改当前 Block 的 `Children[]`。如果拖拽的是 Block，则移动整个子树；如果拖拽的是 Toolpath，则只移动该运动叶子。

## 8. 自动排序

自动排序由 ToolpathOrderService 承载，Facades 只负责命令入口、开启 undo step 和写回排序结果。

输入：

```text
ToolpathOrderRequest
  BlockEntityID
  Strategy
  Options
  IncludeNestedBlocks
```

输出：

```text
ToolpathOrderPlan
  BlockEntityID
  Strategy
  OrderedChildren[]
  EstimatedCost
  CostBreakdown
  Diagnostics[]
```

MVP 策略：

```text
NearestNeighbor
```

考虑因素：

- 当前刀路结束点到下一刀路下刀点的距离。
- 下一刀路是否反向。
- 下刀点位置。
- 姿态变化成本。
- 普通空移和蛙跳成本。

评分：

```text
cost =
  airMoveDistanceWeight * distance
  + postureChangeWeight * postureDelta
  + liftPenalty
  + frogJumpPenalty
```

ToolpathOrderPlan 的 `OrderedChildren[]` 只表达 Block.Children 的目标顺序，不给 Path 写 Order 字段。

## 9. 与运动规划关系

顺序规划输出的是 Program Block 树中的逻辑加工顺序。

运动规划会进一步生成：

- 切割段。
- 空移段。
- 蛙跳段。
- 姿态过渡。

顺序规划不直接生成关节运动。

## 10. 撤销还原

应支持撤销还原：

- 创建 CuttingLayer。
- 删除 CuttingLayer。
- 重命名 CuttingLayer。
- 修改 CuttingLayer 的 CuttingProcessID。
- 创建 VisibleLayer。
- 删除 VisibleLayer。
- 重命名 VisibleLayer。
- 修改 VisibleLayer 颜色。
- 移动刀路到 CuttingLayer。
- 移动刀路到 VisibleLayer。
- 修改工艺。
- 修改顺序。
- 修改刀路方向。

## 11. 测试点

- 创建项目后存在默认 CuttingLayer 和默认 VisibleLayer。
- 刀路不能缺少 CuttingLayer 或 VisibleLayer。
- 锁定 VisibleLayer 后不能通过前端交互修改该层刀路。
- 微联参数越界会被拒绝。
- 自动排序能输出完整刀路顺序。
- 手动拖拽后 Block.Children 持久化。
