# 06 特征识别与手动拾取详细设计

## 1. 模块定位

特征识别与手动拾取模块负责从目标工件 BRep 中得到参数化特征，并支持用户手动拾取 edge/loop 作为人工输入。

MVP 支持：

- 自动识别孔。
- 自动识别坡口孔，包括 V 坡口和 X 坡口。
- 手动拾取 edge。
- 手动拾取 loop。

不支持：

- 手动画线生成刀路。
- STL 目标工件拾取。

## 2. 模块边界

FeatureRecognitionService：

- 输入 BRep 模型资源和特征定义。
- 识别孔、坡口孔、外轮廓等参数化特征。
- 输出 `RecognizedFeature[]`，不直接创建 Path，不修改 Repository。

PickService：

- 接收前端拾取请求。
- 结合拾取过滤器确定 edge/loop。
- 更新当前选择。

ToolpathGenerationService：

- 消费参数化特征生成刀路曲线数据。
- 不在本模块内实现。

## 3. 数据结构

特征识别输入：

```text
FeatureRecognitionRequest
  BRep
    BRepResourceID
    ExpectedDataVersion
    Options
  FeatureDefinitions[]
  Scope
  Options
  ReturnPreviewCurves
```

```text
FeatureDefinition
  Kind              // Hole / BevelHole / OuterLoop / OpenEdge / Custom
  Parameters        // 半径范围、坡口角范围、深度范围、容差等
```

特征识别输出：

```text
FeatureRecognitionResult
  DataVersion
  Features[]
  Diagnostics[]
```

```text
RecognizedFeature
  CandidateID
  Kind
  Role
  Confidence
  Parameters
  PreviewCurves[]
  SourceRefs[]
  Warnings[]
```

孔特征的参数化表达示例：

```text
RecognizedFeature.Kind = Hole
Parameters
  Path              // 世界坐标闭合曲线或可转曲线的参数表达
  Depth
  Axis
  Radius
  Center
  Direction
```

坡口孔补充：

```text
Parameters
  BevelType         // V / X
  BevelAngle
  UpperPath
  LowerPath
```

```text
CurrentSelection
  HoverKind
  HoverID
  SelectedKind
  SelectedID
  SelectedLabel
```

当前 hover 不持久化。已确认刀路不依赖 RecognizedFeature 或 BRep 继续存在；它必须能仅凭 PathCurveResource 和 PathEntity 恢复。

## 4. 自动识别流程

```text
1. CommandHandler 根据 Workpiece 找到 BRepResourceID
2. 构造 FeatureRecognitionRequest
3. FeatureRecognitionService 读取 BRep 模型
4. 按 FeatureDefinition 遍历面、边、环和几何关系
5. 识别孔、坡口孔等参数化特征
6. 生成 RecognizedFeature
7. 返回 FeatureRecognitionResult
8. 可选把结果作为 RuntimeOnly 缓存资源供前端选择
```

## 5. 孔识别原则

孔候选通常满足：

- loop 闭合。
- 位于工件表面。
- 几何连续。
- 可形成切割边界。

坡口孔补充判断：

- loop 周边存在倾斜面。
- 两侧倾斜面可能构成 V 坡口。
- 双侧坡口可能构成 X 坡口。

MVP 不要求算法完美，但必须保留手动拾取兜底。

## 6. 手动拾取流程

```text
1. 前端读取鼠标位置和拾取过滤器
2. 前端通过输入 PDO 或命令提交拾取请求
3. PickService 查询 BRep 拾取索引或识别结果缓存
4. 返回命中的 edge/loop
5. Repository 更新 CurrentSelection
6. 前端高亮当前选择
```

## 7. 拾取过滤器

```text
PickFilter
  AllowEdge
  AllowLoop
  AllowHoleCandidate
  AllowBevelHoleCandidate
```

UI 应允许用户切换 edge/loop 过滤器。

## 8. 与刀路的关系

参数化特征只作为刀路生成输入。

```text
RecognizedFeature -> ToolpathGenerationService -> PathCurveResource -> PathEntity
```

刀路生成后，Path 可以保存 SourceFeature 快照，但不得依赖 CAD topo 或识别缓存作为主身份。

## 9. 失败处理

- 未导入模型：识别失败。
- 几何内核缓存不可用：识别失败。
- 自动识别为空：返回空 Features，不算错误。
- 拾取不到对象：返回 Miss。
- 拾取对象已失效：要求刷新候选列表。

## 10. 测试点

- 导入含孔 STEP 后能识别 Hole 参数化特征。
- 识别为空时仍能手动拾取。
- edge 过滤器只命中 edge。
- loop 过滤器只命中 loop。
- 生成刀路后删除识别缓存不影响刀路显示和保存。
