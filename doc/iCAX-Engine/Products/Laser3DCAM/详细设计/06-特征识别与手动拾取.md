# 06 特征识别与手动拾取详细设计

## 1. 模块定位

特征识别与手动拾取模块负责从目标工件中得到候选切割对象。

MVP 支持：

- 自动识别孔。
- 自动识别坡口孔，包括 V 坡口和 X 坡口。
- 手动拾取 edge。
- 手动拾取 loop。

不支持：

- 手动画线生成刀路。
- STL 目标工件拾取。

## 2. 模块边界

FeatureRecognitionService：

- 读取目标工件几何。
- 识别候选 edge/loop。
- 输出候选对象列表。

PickService：

- 接收前端拾取请求。
- 结合拾取过滤器确定 edge/loop。
- 更新当前选择。

ToolpathGenerationService：

- 消费候选对象生成刀路。
- 不在本模块内实现。

## 3. 数据结构

```text
PickableObject
  PickableID
  Kind(Edge/Loop)
  Label
  WorkpieceID
  SourceTopoRef
  BoundingBox
  Role(Hole/BevelHole/Manual/Unknown)
  Confidence
```

```text
RecognitionResult
  WorkpieceID
  Version
  PickableObjects[]
  Warning[]
```

```text
CurrentSelection
  HoverKind
  HoverID
  SelectedKind
  SelectedID
  SelectedLabel
```

当前 hover 不持久化。已确认刀路不依赖 PickableObject 继续存在。

## 4. 自动识别流程

```text
1. 读取 WorkpieceComponent
2. 获取运行时 BRep/TopoIndex
3. 遍历边、环和面
4. 根据几何和拓扑判断孔特征
5. 判断坡口类型
6. 生成 PickableObject
7. 返回候选列表
8. 可选保存识别结果缓存
```

## 5. 孔识别原则

孔候选通常满足：

- loop 闭合。
- 位于工件表面。
- 几何连续。
- 可形成切割边界。

坡口孔补充判断：

- loop 周边存在倾斜面。
- 两侧倾斜面可能构成 V 坡口。
- 双侧坡口可能构成 X 坡口。

MVP 不要求算法完美，但必须保留手动拾取兜底。

## 6. 手动拾取流程

```text
1. 前端读取鼠标位置和拾取过滤器
2. 前端通过输入 PDO 或命令提交拾取请求
3. PickService 查询 PickableObject 或运行时 topo index
4. 返回命中的 edge/loop
5. Repository 更新 CurrentSelection
6. 前端高亮当前选择
```

## 7. 拾取过滤器

```text
PickFilter
  AllowEdge
  AllowLoop
  AllowHoleCandidate
  AllowBevelHoleCandidate
```

UI 应允许用户切换 edge/loop 过滤器。

## 8. 与刀路的关系

候选对象只作为刀路生成输入。

```text
PickableObject -> ToolpathCurveResource -> ToolpathEntity
```

刀路生成后，Toolpath 可以保存 OptionalSourceInfo，但不得依赖 CAD topo 作为主身份。

## 9. 失败处理

- 未导入模型：识别失败。
- 几何内核缓存不可用：识别失败。
- 自动识别为空：返回空列表，不算错误。
- 拾取不到对象：返回 Miss。
- 拾取对象已失效：要求刷新候选列表。

## 10. 测试点

- 导入含孔 STEP 后能识别候选 loop。
- 识别为空时仍能手动拾取。
- edge 过滤器只命中 edge。
- loop 过滤器只命中 loop。
- 生成刀路后删除识别缓存不影响刀路显示和保存。
