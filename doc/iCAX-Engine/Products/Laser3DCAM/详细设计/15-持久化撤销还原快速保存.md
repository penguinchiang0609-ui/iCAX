# 15 持久化、撤销还原与快速保存详细设计

## 1. 模块定位

本模块定义三维线条切割 CAM 项目文件保存、撤销还原和快速保存的产品级使用方式。

底层能力由 Database、Resources 和项目文件模块提供，产品模块负责正确使用这些能力。

## 2. 项目文件内容

项目文件应包含：

```text
FileHeader
  Magic = ICAX_LASER_3D_CAM
  FormatVersion
  SchemaVersion

DatabaseSnapshot
ResourcePackage
Index
```

## 3. 必须持久化

- ProjectContext settings。
- WorkpieceComponent。
- RawCadResource。
- BRepModelResource。
- ToolAssemblyComponent。
- ToolDescriptionResource。
- ToolComponentModelResource。
- CollisionGeometryResource。
- LayerComponent。
- ToolpathComponent。
- PathCurveResource。
- ToolpathProcess。
- OrderPlan。
- PoseFieldResource。
- MotionPlanResource。
- SafetyCheckResult。

## 4. 不持久化

- hover。
- 仿真播放游标。
- 前端面板状态。
- 前端相机。
- 几何内核运行时指针。
- WebGL 对象。
- Jolt body 指针。

## 5. 资源内嵌规则

Database 中只保存 ResourceID。

Resources 中保存：

- 原始二进制。
- hash。
- 原始路径。
- 元数据。

重新打开项目时：

- 不要求原始外部文件存在。
- ResourceID 必须能解析到内嵌资源。
- Runtime cache 可重建。

## 6. 撤销还原

应支持撤销还原：

- 导入或切换切割头。
- 导入目标工件。
- 生成或删除刀路。
- 修改图层。
- 修改刀路工艺。
- 修改刀路顺序。
- 生成或更新姿态场。
- 生成或更新运动规划。
- 更新安全检查结果。

不支持撤销还原：

- hover。
- 仿真播放位置。
- 临时视图状态。

## 7. 撤销还原边界

产品命令应使用明确的 begin/end 包围用户操作。

示例：

```text
BeginUndoRedoStep("Generate toolpath")
  Add PathCurveResource
  Add PathEntity
EndUndoRedoStep()
```

事务和撤销还原不能绑定。

- 事务用于保证一组数据修改成功或失败。
- 撤销还原用于把用户操作记录为一个可回退 step。

## 8. 快速保存

快速保存依赖 Database 操作日志。

产品要求：

- 每次主数据修改都应经过 Repository。
- 不允许绕过 Repository 修改持久化主数据。
- Resource 变更必须与 Database 引用保持一致。

资源新增时：

```text
1. 写入 ResourcePool
2. 写入 Database 引用
3. 操作日志记录 Database 修改
4. 保存点把新增资源纳入资源包
```

## 9. 保存流程

```text
1. 暂停项目写入或进入保存快照阶段
2. 导出 DatabaseSnapshot
3. 收集 Database 引用的 ResourceID
4. 打包 ResourcePackage
5. 写入 FileHeader
6. 写入项目文件
7. 清理或截断 quick save log
```

## 10. 打开流程

```text
1. 读取 FileHeader
2. 校验 magic
3. 根据 magic 选择产品
4. 加载 DatabaseSnapshot
5. 加载 ResourcePackage
6. 重建 runtime cache
7. 如存在 quick save log，执行恢复
8. 创建 Project 和 MainScene
```

## 11. 一致性检查

打开项目后应检查：

- Database 引用的 ResourceID 都存在。
- Toolpath 引用的 CurveResource 存在。
- ToolAssembly 引用的 ToolDescriptionResource 存在。
- Workpiece 引用的 RawCadResource、BRepModelResource、TopologyResource 存在。
- MotionPlan 引用的 Toolpath 存在。

## 12. 测试点

- 保存后删除外部 STEP/SDF/STL，项目仍可打开。
- 生成刀路后撤销能删除刀路和曲线引用。
- 重做能恢复刀路。
- 保存后 quick save log 被清空或截断。
- 崩溃恢复能基于项目文件和 quick save log 恢复到最近状态。
