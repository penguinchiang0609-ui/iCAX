# 17 工件区意图刀路衍生模型

## 1. 文档目的

本文定义工件区中意图刀路的组织、衍生、更新和交互模型，覆盖以下长期需求：

- 从 CAD 特征识别、拓扑拾取或手工绘制创建意图刀路。
- 对意图刀路增加微联、引入、引出、桥接等工艺。
- 对完整刀路或内部子刀路执行打断、合并、删除和再次加工。
- 上游刀路或工艺参数变化后，下游衍生结果自动更新。
- 合并、打断取消时尽量恢复原子刀路；原子刀路不存在时保留当前结果。
- 使用 TreeView 向用户展示当前可理解、可选择的加工结构。
- 加工顺序可以跨工件，同时保留零件归属及生产计数依据。

本文中的“刀路”均指工件区的机床无关意图刀路，不是作业区生成的切割头实际运动轨迹，也不是最终 NC 插补轨迹。

## 2. 核心结论

工件区采用“意图刀路衍生模型”：

1. 意图刀路是用户编辑和选择的核心对象。
2. 一个意图刀路可以通过工艺生成若干子意图刀路，形成可展开的衍生组。
3. 微联等内部工艺生成的子刀路仍属于母刀路，不能直接参与全局排序。
4. 合并、打断生成新的独立意图刀路，不把原子刀路从原母刀路中搬走。
5. 被合并或打断消费的原子刀路继续存在，但被当前衍生结果替代，不再重复输出加工任务。
6. 衍生结果保存到输入意图刀路的来源关联，上游变化时自顶向下重新生成。
7. TreeView 展示当前所有权和生成分组；多输入来源关系不强行显示成树。
8. 当前有效加工输出由替代关系推导，不能简单等同于所有叶子节点。

该模型不采用“把选中加工段移动给新刀路”的所有权转移方案。合并、打断均采用复制衍生语义，原结构得以保留并继续参与依赖更新。

## 3. 意图刀路

意图刀路描述用户希望加工的几何范围、方向和工艺含义。它可以表示：

- 一个识别出来的孔轮廓。
- 一条拾取的 CAD 边或轮廓链。
- 一条手工绘制的切割线。
- 微联前后的切割段。
- 微联区间。
- 引入或引出段。
- 桥接段。
- 由多个输入合并得到的连续刀路。
- 由一条刀路打断得到的独立刀路。

意图刀路拥有稳定 Entity 身份。空间曲线、参数范围和工艺参数可以重新计算，但只要衍生角色仍然能够匹配，Entity 身份应保持稳定，使其下游工艺可以继续关联。

## 4. 母刀路与子刀路

工艺可以把一条意图刀路展开成若干子意图刀路。

例如给刀路 A 增加一个微联 M1：

```text
刀路 A
├─ 切割刀路 A1
├─ 微联刀路 M1
└─ 切割刀路 A2
```

如果继续给 A2 增加微联 M2：

```text
刀路 A
├─ 切割刀路 A1
├─ 微联刀路 M1
└─ 切割刀路 A2
   ├─ 切割刀路 A21
   ├─ 微联刀路 M2
   └─ 切割刀路 A22
```

母子关系表达当前生成结构和编辑层次：

- 默认场景单击内部任意段时，选择当前编辑层级中的母刀路。
- 双击母刀路进入下一层，才可以直接选择它的子刀路。
- 有子刀路的母刀路不与子刀路重复输出加工任务。
- 母刀路整体可以参与所在层级的排序，内部子刀路只能在母刀路内部调整。
- 如果内部子刀路需要独立参与全局排序，必须通过“提升为独立刀路”或“打断为独立刀路”生成新的顶层结果。

## 5. 意图刀路来源

每条意图刀路必须说明自己从哪里产生。来源是关联和重算依据，不是所有权关系。

按照 EC 设计，来源在概念上属于同一族，但实现上使用不同组件组合，不建立 C++ 组件继承树。

### 5.1 CAD 来源

用于特征识别、拓扑拾取和 CAD 轮廓提取，至少保存：

- 工件 Entity。
- CAD 特征编码。
- 拓扑类型和拓扑标识。
- 几何指纹。
- 在来源曲线或轮廓上的参数范围。
- 当前关联状态和匹配置信度。

CAD 来源负责随 CAD 修改重新查找对应特征。无法唯一匹配时不得静默选择候选，应将意图刀路标记为需要复核或来源失效。

### 5.2 意图刀路来源

用于表达一条刀路由其他意图刀路衍生而来，至少保存：

- 一个或多个输入意图刀路。
- 生成操作或工艺标识。
- 输出角色。
- 来源参数范围或对应特征。
- 当前关联状态。

微联通常只有一个输入；合并和桥接允许多个输入。因此完整来源关系可以形成有向无环图，但它不直接进入左侧所有权树。

### 5.3 手工绘制来源

用于用户凭空绘制的加工几何，至少保存：

- 绘制坐标系或承载曲面。
- 草图或曲线资源。
- 创建工具及关键约束。

手工绘制刀路不依赖 CAD 特征，但仍可被后续微联、合并和打断操作引用。

### 5.4 构造来源

用于生成输入几何中原本不存在的新路径，例如：

- 桥接。
- 延长。
- 圆角或倒角过渡。
- 偏置。
- 投影。
- 引入和引出。

构造来源保存输入意图刀路、构造类型和构造参数。

### 5.5 模板或复制来源

用于把一个零件的加工方式应用到其他相似零件，保存：

- 原意图刀路或原加工模板。
- 目标工件实例。
- 特征对应关系。
- 几何映射和参数覆盖。

它用于表达可继续更新的加工方法实例，不等同于无关联的普通复制。

### 5.6 外部导入来源

用于导入 DXF、外部 CAM 轮廓或其他已包含加工含义的数据，保存外部文件、外部对象标识、导入坐标系和必要的内容指纹。

## 6. 工艺生成与输出角色

工艺不是对曲线数组的一次性破坏，而是接收意图刀路并生成子意图刀路的参数化操作。

微联 M1 的逻辑输出不是匿名的 `Children[0..2]`，而是：

```text
BeforeMicroJoint(M1)
MicroJoint(M1)
AfterMicroJoint(M1)
```

输出角色必须稳定。移动微联后，系统依靠“工艺标识 + 输出角色”把旧子刀路对应到新结果：

```text
旧 A1 -> 新 BeforeMicroJoint(M1)
旧 M1 -> 新 MicroJoint(M1)
旧 A2 -> 新 AfterMicroJoint(M1)
```

由此，A2 上继续添加的微联和其他工艺可以保留下来并继续向下更新。

其他操作采用相同原则：

```text
引入：LeadIn、MainCut
引出：MainCut、LeadOut
打断：BeforeBreak、AfterBreak
桥接：FirstInput、Bridge、SecondInput
合并：MergedResult，内部保留输入对应关系
```

## 7. 自顶向下更新

上游 CAD、意图刀路或工艺参数变化后，从变化节点开始依次重新计算下游：

```text
CAD或母刀路变化
  -> 重算当前工艺
  -> 按输出角色匹配已有子刀路
  -> 更新子刀路几何和参数范围
  -> 继续重算子刀路上的下游工艺
```

例如移动上层微联 M1：

```text
A 更新
  -> A1、M1、A2 更新
  -> A2 更新触发 M2 重算
  -> A21、M2、A22 更新
```

更新必须遵循以下规则：

1. 能通过稳定输出角色唯一匹配时，保留原 Entity 身份。
2. 输出数量变化但能够建立唯一对应时，更新现有节点并创建或删除必要节点。
3. 无法可靠对应时，不得把旧的下游工艺静默挂到新段上。
4. 无法更新的节点保留用户参数并标记“上游来源失效”，等待重新定位、解除关联或删除。
5. 已被替代的原子刀路仍然参与依赖更新，只是不参与有效加工输出。

## 8. 替代关系与有效输出

合并和打断不会从原母刀路中搬走原子刀路，而是生成独立结果，并使输入原子刀路被当前结果替代。

例如：

```text
刀路 A
├─ A1
├─ 微联 MA
└─ A2［被 C 替代］

刀路 B
├─ B1［被 C 替代］
├─ 微联 MB
└─ B2

刀路 C［合并结果］
├─ A2 的衍生结果
└─ B1 的衍生结果
```

“被替代”不等于几何过时：

- 原节点仍然存在。
- 原节点仍然跟随上游变化而更新。
- 原节点仍然可以作为下游衍生结果的输入。
- 原节点不再独立产生加工任务，避免重复加工。

建议将以下状态分开：

- `Active`：当前有效并参与加工输出。
- `Superseded`：由其他当前有效结果替代，不独立输出。
- `SourceInvalid`：来源无法重新计算或关联。
- `Deleted`：用户明确删除。

替代状态应优先根据有效衍生结果的输入关系推导。衍生结果记录自己的输入，系统运行时构建“由谁替代”的反向索引，避免两侧持久化产生冲突。

同一加工方案中，一个原子意图刀路默认只能被一个当前有效衍生结果消费。若用户需要从同一输入建立多个并行方案，应先复制刀路或创建独立加工方案。

## 9. 合并

合并接收两个或多个意图刀路，生成新的独立意图刀路：

```text
输入：A2、B1
输出：合并刀路 C
```

合并后：

- A2、B1继续留在原母刀路中。
- A2、B1被 C 替代，不再独立输出。
- C拥有自己生成的子刀路，而不是直接取得A2、B1的所有权。
- C记录到A2、B1的关联及相应几何区间。
- A2或B1随上游更新时，C继续重算。

输入首尾不连续时，不得把结果伪装成几何合并。用户必须明确选择创建桥接，结果表达为：

```text
连续刀路 C
├─ 第一输入的衍生段
├─ 桥接段
└─ 第二输入的衍生段
```

## 10. 打断

需要区分两种操作。

### 10.1 内部分段

内部分段仍然保留母刀路作为整体，结果作为子刀路：

```text
刀路 A
├─ 前段
└─ 后段
```

前后段不能独立参与全局排序。

### 10.2 打断为独立刀路

打断为独立刀路时生成新的顶层结果：

```text
原子刀路 A0［被打断结果替代］

打断结果组
├─ 独立刀路 A1
└─ 独立刀路 A2
```

A1、A2引用A0作为来源；A0仍留在原母刀路中并继续更新，但不重复输出。

## 11. 取消合并与取消打断

取消衍生操作时，系统按每个输入分别处理。

### 11.1 原子刀路仍存在

如果原子刀路仍存在且没有被其他当前结果消费：

1. 移除或停用当前合并、打断结果。
2. 原子刀路不再被该结果替代。
3. 原子刀路恢复为当前有效输出。

### 11.2 原子刀路已经删除

如果原子刀路已经不存在，取消操作不得丢失当前可见加工结果：

1. 从合并或打断结果中提取相应结果刀路。
2. 将其固化并提升为独立意图刀路。
3. 解除对缺失原子刀路的实时依赖。
4. 保留来源说明，用于追溯，但不再要求来源存在。

### 11.3 部分输入存在

对每个输入独立处理：

- 能恢复的原子刀路恢复有效。
- 无法恢复的结果部分成为独立刀路。
- 不因一个输入缺失而丢弃其他输入的有效结果。

### 11.4 原子刀路又被其他结果消费

如果原子刀路仍存在，但已经由另一个当前有效结果替代，则不能直接恢复为 Active，否则会重复加工。系统应保留拆出的当前结果或提示用户选择保留哪一个替代关系。

## 12. 删除语义

删除操作必须区分对象类型：

- 删除母刀路：删除其当前拥有的生成树；引用它的外部衍生结果进入来源失效，不自动级联删除。
- 删除被替代的原子刀路：引用它的衍生结果继续保留当前几何，但进入来源失效或等待固化状态。
- 删除衍生结果：解除它造成的替代关系；仍存在且未被其他结果消费的输入恢复有效。
- 删除 CAD 特征或工件：意图刀路保留，但 CAD 来源进入失效状态。

系统不得仅因来源被删除就自动删除用户已经编辑过的衍生刀路。

## 13. 固化与解除关联

默认衍生结果保持来源关联并随上游更新。用户可以主动执行“固化”或“解除来源关联”：

```text
固化前：上游变化 -> 当前刀路自动更新
固化后：当前几何成为独立意图刀路，上游变化不再影响它
```

固化后仍可保留只读来源说明，但来源不再作为重新生成输入。

固化用于表达用户明确认可当前结果并希望从此独立编辑，不是错误恢复或临时兼容机制。

## 14. 工件归属与生产统计

加工组织不能以工件为唯一父级，因为实际加工顺序可以跨工件。但每条意图刀路必须记录自己服务的工件实例。

普通刀路通常关联一个工件；共边或跨工件桥接可以关联多个工件。工件归属至少包含：

- 工件实例。
- 归属角色，例如主要加工、共享加工、桥接过渡或分离切割。
- 是否属于该工件完成判定的必要项。

意图刀路进入作业区并生成实际运动和 NC 动作时，工件归属与来源标识必须继续传递。生产计数依据工件实例的必要加工项完成状态计算，不按照执行刀数直接累计。

## 15. 左侧 TreeView

左侧 TreeView 展示当前生成结构，不展示完整来源 DAG。

默认结构：

```text
意图刀路
├─ 刀路 A
│  ├─ 切割段 A1
│  ├─ 微联 M1
│  └─ 切割段 A2［已被刀路 C 替代］
├─ 刀路 B
│  ├─ 切割段 B1［已被刀路 C 替代］
│  └─ 切割段 B2
└─ 刀路 C［合并结果］
   ├─ A2 衍生段
   └─ B1 衍生段
```

默认可以隐藏已被替代节点；开启“显示来源和已替代刀路”后，以灰色和替代者标识显示。

场景选择规则：

- 单击：选择当前编辑层级中的整体刀路。
- 双击：进入所选刀路内部，允许选择子刀路。
- `Esc`或双击空白：退出一层。
- 顶层刀路可以参与全局排序。
- 内部子刀路只能在当前母刀路内排序。
- 需要跨出母刀路排序时，必须生成独立刀路。

来源信息在属性面板展示：

```text
生成方式：合并
输入来源：刀路 A / A2、刀路 B / B1
关联状态：有效
当前替代：A2、B1
```

## 16. 数据关系总结

系统同时维护四种正交关系：

### 16.1 当前生成树

决定：

- TreeView如何展开。
- 默认场景选择谁。
- 谁是母刀路、谁是内部子刀路。
- 内部局部顺序。
- 删除母节点时处理哪些子节点。

### 16.2 来源依赖

决定：

- 刀路从CAD、其他意图、绘制或构造中的何处产生。
- 上游变化时需要重新计算哪些下游。
- 合并、桥接等多输入关系。
- 来源失效和重新关联。

### 16.3 替代关系

决定：

- 哪些原子刀路暂时不独立输出。
- 取消合并或打断时哪些原子刀路可以恢复。
- 如何避免原刀路和衍生结果重复加工。

替代关系优先由当前有效衍生结果的输入来源推导。

### 16.4 工件归属

决定：

- 一刀服务哪些工件。
- 如何按工件查看进度。
- 共边、跨工件桥接如何统计。
- 作业和NC执行后如何计算零件完成数量。

## 17. 关键不变量

实现必须始终维护：

1. 当前生成树中一个子节点只能有一个直接母节点。
2. 当前生成树不允许形成环。
3. 来源依赖允许多输入，但不允许形成依赖环。
4. 有子刀路的母刀路不与其有效子刀路重复输出。
5. 被当前有效结果替代的原子刀路不独立输出。
6. 一个原子刀路在同一方案中默认只能被一个当前有效结果消费。
7. 已被替代不等于来源失效；已被替代节点仍继续更新。
8. 重新生成时优先通过稳定输出角色保持Entity身份。
9. 无法可靠映射时必须显式失效，不得静默错绑。
10. 取消衍生操作不得因为来源缺失而丢失当前加工几何。
11. 删除来源默认不级联删除已产生的外部衍生结果。
12. 工件归属必须从意图刀路贯穿到作业轨迹和NC执行结果。

## 18. 与现有实现的关系

现有 `CProgramNodeComponent`、`CPathComponent` 和 `CBlockComponent` 以程序节点、Block和Path为核心，无法完整表达本模型中的：

- 多种意图刀路来源。
- 多层工艺衍生。
- 稳定输出角色。
- 替代关系与有效输出推导。
- 多输入合并。
- 跨工件归属。
- 来源失效、固化和取消衍生。

新产品无需兼容现有数据版本，后续实现应按照 EC 组合原则重新拆分组件，不继续通过 `CProgramNodeComponent` 继承扩展，也不把来源、工艺、几何、归属和执行结果继续集中到单个 `CPathComponent` 中。

本文先确定领域语义。具体组件字段、Repository过滤器、命令事务和前端PDO结构在实现设计阶段基于这些不变量进一步细化。
