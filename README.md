# CAX

## 概述

### 概念

#### CAD

定位是纯数学几何意义，其中二维图元支持绘制、编辑。三维实体为借助二维图元使用四大造型算法得到。三维实体可以CSG树结构装配成几何实体树GeometryTree，且可添加约束或使用参数脚本表达，从而支持几何实体树的快速个性化实例。

此处的基础理论：所有三维模型都可通过对毛坯采用切削钻铣以及焊接得到。

四大造型算法：

- 拉伸：方向、高度、角度进行拉伸，可以拉伸出椎体。Extrude
- 旋转体：待旋转图元，旋转轴。旋转轴与待旋转图元所在平面不能垂直，否则无法生成三维图元。Revovle
- 扫掠：图元沿着路径扫掠得到实体或曲面。Sweep
- 放样：沿一系列横截面来创建新的实体或曲面。Loft

为避免歧义，拉伸不支持沿路径拉伸（划入扫掠），扫掠不支持扭曲扫掠（划入放样）。

CAD中支持直接绘制三维曲线。

渲染交互层采用OSG。



#### 拆单

职责：构建产品，输出产品效果图（用于确认与展示）、产品爆炸图装配图（用于安装）、工件清单（包括几何信息）（用于生产）、连接件清单（俗称五金）（用于采购）。

- 零件

  组成产品的最小部件。零件几何信息由CAD输出的几何实体树表达。

- 装配体

  使用零件以及组装工艺组装而成。可对装配体添加约束或使用脚本参数表达，从而形成可定制化装配体。

- 产品

  使用零件、装配体使用组装工艺组装而成。可对产品添加约束或使用脚本参数表达，从而形成产品模板，方便后续快速得到个性化产品实例。

> [!NOTE]
>
> 构件产品的零件与输出的零件不等价，如两个薄板零件之间的连接件选择为折弯，则生产时，两个零件会合为一个零件，连接件选择为焊接时则生产时为两个零件。而且原始零件与待加工零件在几何上也并非等价，如使用咬口工艺，待加工零件会比原始零件多一部分用于生成咬口。原始零件是为产品设计服务，待加工零件是面向生产，因其是确定的，所以其必须为个性化实例。

第一阶段不包含。



#### CAPP 生产计划

定位：根据待加工零件信息，结合现状（工艺路线、材料等等，即实际生产条件），输出每个待加工零件的工艺路线图（即如何加工出来，分解为一个个工序，并为每个工序输出加工信息，每一个工序的加工信息可等价理解为该工序的工件）。如一个台盆分解为开料出一个原型石柱工件，再基于石柱工件车出一个精加工的台盆工件。

此处的核心在于基于零件的特征信息，结合实际条件，自动规划生产过程，并为每个生产过程输出工艺参数，如开料的进给速度、主轴速度、下刀深度，车铣的主轴转速，吃刀深度等等。



此处需要深入理解CAPP在现有生产中的使用，此处可能是整个产品最吸引终端用户的点。因其的核心在于针对零件，抽取零件特征，制定工艺路线，如在激光开料环境下，五角星坡口工件应该使用哪些具体的工艺。其核心在于智能化制定工艺路线（即都使用哪些工艺，顺序）。



从大的范畴，其属于CAM的一部分。



#### 仓储

职责：原材、余料的信息管理，包括几何形状、材质、数量等。

原材或余料的几何形状由性化实例的三维图元的CSG树表达。



#### 工件

职责：某工位需要加工成的目标产品信息，其几何信息由性化实例的三维实体CSG树表达。

工件与CAM密切相关，与产品本来的零件已经脱钩。

提供工件的设计功能，即工件的来源可以是CAPP的输出物，也可以是本地直接绘制的。



#### CAM

职责：数控编程工具，服务于CNC。其实现思路为根据零件的几何信息，原材的材料信息，机器配置等，输出加工指令（加工指令统一以g代码表达）。

在CAM缺失的情况下，数控人员可以进行手动编程（g代码编写）来驱动CNC工作。但手动编程的难度大，尤其对于复杂零件，即CAM的价值在于解放数控编程人员。

核心职责：

- 刀路：根据零件几何信息，生成刀路，并支持刀路的进一步编辑、合并、分组、排序等等。

- 工艺：给刀路附着加工工艺信息，工艺的添加有顺序关系。

  备注：刀路组自身为空对象，其上可以附加工艺，如用于桥切机器中的吸盘

- 后处理：根据机器配置，输出加工指令。后处理为CAM的插件模块，即可针对不同的机器输出不同的加工指令。

- 仿真：刀路指令仿真，刀具碰撞测试等。（务必注意，此处的仿真与CAE属于不同范畴）

开料机CAM，如板式家具的加工中心或电锯中心等会在以上的职责中插入排样模块，负责根据将零件在原材上进行有规律的排布，然后生成加工刀路。

CAM中支持直接以CAD方式绘制刀路，但不支持绘制曲面或实体。（某些情形下，会有一些零件以外的刀路，如开料机中的余料线）



#### CNC

职责：根据加工指令驱动机器对原料进行真实加工，得到目标工件。

由CNC与PLC构成。CNC负责运动控制（轴），PLC负责逻辑控制（其他设备，如开关等）。

其中CNC与PLC存在大量交互，CNC以寻址方式暴露数据信息给PLC，PLC同样以寻址方式暴露信息至CNC。此处将寻址部分抽取作为公共基础部分，即CNC与PLC

操作的为同一片区寻址。定时分片机制，1ms循环一次，每次先处理CNC再处理PLC。

CNC的处理包括G代码解析，结合设备状态信息进行运动规划生成运动指令，PLC读取设备状态信息、运动指令进行逻辑运算，生成逻辑控制指令。然后统一下发至硬件，由硬件执行。



### 核心数据结构

#### MATH库

data/geometry 定义一系列基础的几何结构。目标为图形的数据结构需要对TRS以及边界友好。
